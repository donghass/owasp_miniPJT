# 미완성/제한 사항 정리

## 1) 정책상 직접 구현 불가 항목

다음은 실제 악용 가능한 공격 코드 또는 의도적 취약점 삽입을 수반하므로 직접 구현하지 않았습니다.

- SQL Injection을 의도적으로 유발하는 쿼리 연결 코드
- Command Injection이 가능한 시스템 명령 실행 엔드포인트
- Path Traversal이 가능한 파일 접근 로직
- 쿠키 변조로 권한 상승이 즉시 가능한 인증 우회 코드
- 무제한/무검증 파일 업로드 취약 로직

## 2) 대신 제공한 범위

- 정상 동작하는 3티어 웹앱 골격 완성
- 관리자 페이지 포함 필수 기능 구현
- 감사 로그 기반의 추적 가능 구조 제공
- 향후 보안 점검 및 취약점 분석팀이 테스트할 수 있는 표준 CRUD/권한 흐름 제공

## 3) 과제 제출 시 작성 권장 항목

- 취약점 분석팀은 별도 문서에서 다음을 정리 권장:
  - 취약점 이름(OWASP 매핑)
  - 재현 조건
  - 영향 범위
  - 개선안
  - 수정 전/후 비교

## 4) 윤리/운영 가이드

- 로컬 또는 폐쇄망에서만 테스트
- 실제 개인정보 저장 금지
- 외부 공개 환경 배포 금지
